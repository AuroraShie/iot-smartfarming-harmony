# 测试操作指南

## 1. 文档目标

这份文档用于把项目里分散的测试说明整理成一套统一流程，方便快速完成以下两类测试：

- 真实联调：HarmonyOS 前端直接连接 ESP32 网关
- 本地前端联调：HarmonyOS 前端连接 `esp-mock-server.js`

当前推荐优先使用真实联调模式，因为现在 `iottest` 已经按鸿蒙端现有接口适配为本地网关服务。

## 2. 当前项目测试模式

### 模式 A：真实 ESP 网关联调

适用场景：

- 验证 `iottest` 当前网络模块
- 验证 ESP32 对外提供的 HTTP 和 WebSocket 接口
- 验证 HarmonyOS 前端是否能直接消费真实设备数据

关键特征：

- ESP32 连接 Wi-Fi 后，在本机开启网关服务
- 默认端口：`8080`
- 关键接口：
  - `GET /gateway/status`
  - `GET /telemetry/realtime`
  - `GET /devices`
  - `POST /devices/{id}/command`
  - `GET /commands/{requestId}`
  - `ws://<esp-ip>:8080/ws`

### 模式 B：本地 Mock Server 联调

适用场景：

- ESP32 不在手边
- 只想验证 HarmonyOS 前端页面和服务层
- 只想做 UI / 交互 / 页面联调

关键特征：

- 启动 `net/HarmoAgri/tools/esp-mock-server.js`
- 默认端口：`3000`
- 适合前端先行开发，但不代表真实 ESP 行为完全一致

## 3. 测试前准备

### 3.1 环境准备

- 电脑已安装 `PlatformIO`
- ESP32 开发板可正常烧录
- ESP32 与电脑接入同一局域网
- `DevEco Studio` 或对应 HarmonyOS 运行环境可正常启动

- `D:\test111\net\HarmoAgri\entry\src\main\ets\services\GatewayService.ets`

## 4. 真实 ESP 网关联调流程

### 第 1 步：配置 ESP32 Wi-Fi 和网关参数

编辑 `D:\test111\iottest\src\config.h`：

```cpp
#define WIFI_SSID "你的WiFi名称"
#define WIFI_PASSWORD "你的WiFi密码"
#define GATEWAY_ID "esp-gateway-001"
#define GATEWAY_PORT 8080
#define GATEWAY_WS_PATH "/ws"
```

建议先不要改端口，保持 `8080`，这样可以和鸿蒙端当前默认配置保持一致。

### 第 2 步：编译并上传 ESP32

在 `D:\test111\iottest` 目录执行：

```powershell
pio run -t upload
pio device monitor
```

如果上传成功，串口里应能看到类似输出：

```text
Connecting to Wi-Fi SSID: ...
Wi-Fi connected, IP: 192.168.x.x
Gateway is serving at http://192.168.x.x:8080
```

记下这个 `ESP IP`。

### 第 3 步：先独立测试 ESP HTTP 接口

在电脑 PowerShell 执行：

```powershell
Invoke-RestMethod http://<ESP_IP>:8080/gateway/status
Invoke-RestMethod http://<ESP_IP>:8080/telemetry/realtime
Invoke-RestMethod http://<ESP_IP>:8080/devices
```

预期结果：

- `/gateway/status` 返回 `gatewayId`、`wifi.ip`、`uptimeSec`
- `/telemetry/realtime` 返回温湿度、光照、土壤湿度等快照
- `/devices` 返回传感器设备和两个执行器：
  - `pump-001`
  - `growlight-001`

### 第 4 步：测试控制命令

测试水泵：

```powershell
$body = @{
  command = "TURN_ON"
  requestId = "cmd-test-001"
  params = @{ durationSec = 10 }
} | ConvertTo-Json

Invoke-RestMethod http://<ESP_IP>:8080/devices/pump-001/command -Method Post -ContentType "application/json" -Body $body
Invoke-RestMethod http://<ESP_IP>:8080/commands/cmd-test-001
```

测试补光灯：

```powershell
$body = @{
  command = "SET_LEVEL"
  requestId = "cmd-test-002"
  params = @{ level = 4 }
} | ConvertTo-Json

Invoke-RestMethod http://<ESP_IP>:8080/devices/growlight-001/command -Method Post -ContentType "application/json" -Body $body
Invoke-RestMethod http://<ESP_IP>:8080/commands/cmd-test-002
```

### 第 5 步：测试 WebSocket

可用浏览器控制台快速测试：

```javascript
const ws = new WebSocket("ws://<ESP_IP>:8080/ws");
ws.onopen = () => console.log("open");
ws.onmessage = (e) => console.log(e.data);
ws.onerror = (e) => console.log("error", e);
```

预期能看到三类消息：

- `telemetry.update`
- `device.status.changed`
- `command.result`

### 第 6 步：测试 HarmonyOS 前端

在鸿蒙应用网关配置中填写：

- `host`: `<ESP_IP>`
- `port`: `8080`
- `protocol`: `HTTP_WS`
- `wsPath`: `/ws`

然后执行：

1. 点击“测试连接”
2. 观察首页是否拿到实时数据
3. 打开设备页检查 `pump-001` 和 `growlight-001`
4. 下发控制命令，检查状态是否回写

## 5. Mock Server 联调流程

### 第 1 步：启动 Mock Server

在 `D:\test111\net\HarmoAgri\tools` 执行：

```powershell
node esp-mock-server.js
```

默认输出：

```text
ESP mock server running at http://127.0.0.1:3000
```

### 第 2 步：让鸿蒙前端指向 Mock Server

如果是在模拟器里访问电脑本机服务，通常使用：

- `host`: `10.0.2.2`
- `port`: `3000`
- `wsPath`: `/ws`

如果是真机联调，则使用电脑真实局域网 IP。

### 第 3 步：用控制台命令模拟状态变化

常用命令：

```text
status
telemetry
devices
device on pump-001
device off pump-001
set telemetry soilMoisture 29.5
fault gateway on
fault gateway off
```

## 6. 推荐测试顺序

建议顺序如下：

1. 先测 ESP 串口是否正常启动
2. 再测 HTTP 接口
3. 再测 WebSocket 推送
4. 最后测 HarmonyOS 页面联调

这样排查最快，因为能尽早判断问题是出在硬件、网络、接口还是前端。

## 7. 测试完成的判断标准

满足以下几点，可以认为本轮联调基本通过：

- ESP32 成功连上 Wi-Fi
- `GET /gateway/status` 正常返回
- `GET /telemetry/realtime` 正常返回
- `GET /devices` 正常返回
- `POST /devices/{id}/command` 能收到 `accepted`
- `GET /commands/{requestId}` 能查到结果
- WebSocket 能收到 `telemetry.update`
- HarmonyOS 前端能显示实时数据并完成控制闭环
