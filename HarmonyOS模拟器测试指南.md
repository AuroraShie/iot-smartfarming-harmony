# HarmonyOS模拟器 + ESP32 完整测试指南

## 你的测试环境

- **本机电脑**：运行后端服务器
- **DevEco Studio**：运行HarmonyOS模拟器（前端）
- **ESP32设备**：传感器数据采集（IoT端）

---

## 关键知识点

### HarmonyOS模拟器访问本机服务器

在HarmonyOS模拟器中，**不能**使用 `localhost` 或 `127.0.0.1`，需要使用：

```
10.0.2.2
```

这是模拟器访问宿主机（你的电脑）的特殊地址。

---

## 完整测试步骤

### 第1步：启动后端服务器

在你的电脑上打开命令行，执行：

```bash
cd d:/test111/net/HarmoAgri/tools
node esp-mock-server.js
```

服务器启动后会显示：
```
ESP mock server running at http://127.0.0.1:3000
```

**保持这个窗口运行！**

---

### 第2步：获取本机IP地址

打开**新的**命令行窗口，执行：

```bash
ipconfig
```

找到你的WiFi连接，记录IPv4地址，例如：
```
192.168.1.105
```

---

### 第3步：配置前端（HarmonyOS应用）

在DevEco Studio中，找到前端项目的配置文件，修改服务器地址：

**文件位置**：`net/HarmoAgri/entry/src/main/ets/services/GatewayService.ets`

或者在配置文件中修改服务器地址：

```typescript
// 将服务器地址改为
const SERVER_HOST = '10.0.2.2';  // 模拟器访问本机
const SERVER_PORT = 3000;
```

**注意**：
- 在**模拟器**中测试时使用：`10.0.2.2`
- 在**真机**上测试时使用：你的本机IP（如 `192.168.1.105`）

---

### 第4步：配置ESP32

在 `iottest/src/config.h` 中修改：

```cpp
// WiFi配置
#define WIFI_SSID "你的WiFi名称"
#define WIFI_PASSWORD "你的WiFi密码"

// 服务器配置
#define SERVER_HOST "192.168.1.105"  // 你的本机IP（不是10.0.2.2！）
#define SERVER_PORT 3000
```

**重要**：ESP32使用真实的本机IP，不是 `10.0.2.2`！

---

### 第5步：开始测试

**启动顺序**：

1. ✅ 启动后端服务器（第1步）
2. ✅ 启动HarmonyOS模拟器
3. ✅ 运行前端应用
4. ✅ 上传ESP32代码并启动

**测试验证**：

- 服务器控制台显示：收到传感器数据
- HarmonyOS应用显示：实时传感器数据
- ESP32串口显示：数据发送成功

## 创建快速启动脚本

在 `d:/test111` 目录下创建 `start-test.bat`：

```batch
@echo off
echo ========================================
echo   启动IoT智能农业测试环境
echo ========================================

echo.
echo [1/3] 启动后端服务器...
start "ESP32 Server" cmd /k "cd /d d:\test111\net\HarmoAgri\tools && node esp-mock-server.js"

echo.
echo [2/3] 等待服务器启动...
timeout /t 3 /nobreak > nul

echo.
echo [3/3] 获取本机IP地址...
echo.
ipconfig | findstr "IPv4"
echo.

echo ========================================
echo   服务器已启动！
echo ========================================
echo.
echo 请记录上面的IPv4地址
echo.
echo HarmonyOS模拟器使用: 10.0.2.2:3000
echo ESP32使用: 你的本机IP:3000
echo.
echo 按任意键退出...
pause > nul
```

双击这个脚本即可一键启动服务器！

## 常见问题

### Q1: HarmonyOS模拟器无法连接服务器

**检查**：
1. 服务器是否正在运行
2. 是否使用 `10.0.2.2` 作为服务器地址
3. 端口号是否正确（3000）

**测试连接**：
在模拟器的浏览器中访问：
```
http://10.0.2.2:3000/gateway/status
```

如果能看到JSON响应，说明连接正常。

### Q2: ESP32无法连接服务器

**检查**：
1. ESP32和电脑是否在同一WiFi网络
2. 服务器地址是否使用真实的本机IP（如 `192.168.1.105`）
3. 防火墙是否阻止了端口3000

**测试连接**：
在电脑浏览器中访问：
```
http://localhost:3000/gateway/status
```

### Q3: 服务器收不到数据

**检查**：
1. ESP32串口监视器是否显示"发送成功"
2. 服务器控制台是否有错误信息
3. JSON数据格式是否正确

## 调试技巧

### 1. 分步测试

**先测试服务器是否正常**：
```bash
curl http://localhost:3000/gateway/status
```

**再测试ESP32连接**：
查看ESP32串口输出

**最后测试HarmonyOS**：
查看应用是否显示数据

### 2. 查看日志

- **服务器日志**：服务器控制台窗口
- **ESP32日志**：串口监视器
- **HarmonyOS日志**：DevEco Studio的Log窗口

### 3. 网络诊断

**检查端口是否被占用**：
```bash
netstat -ano | findstr :3000
```

**检查防火墙**：
Windows设置 → 防火墙 → 允许应用通过防火墙
