## 快速启动（3步完成）

### 步骤1：启动模拟服务器

打开命令行，执行：

```bash
cd d:/test111/net/HarmoAgri/tools
node esp-mock-server.js
```

启动成功后显示：
```
ESP mock server running at http://127.0.0.1:3000
```

**保持这个窗口运行，不要关闭！**

---

### 步骤2：获取本机IP地址

打开**新的**命令行窗口，执行：

```bash
ipconfig
```

找到你的WiFi连接，记录IPv4地址，例如：
```
IPv4 Address. . . . . . . . . . . : 192.168.1.105
```

---

### 步骤3：配置ESP32代码

在 `iottest/src/config.h` 中修改：

```cpp
// WiFi配置
#define WIFI_SSID "你的WiFi名称"           // 改成你的WiFi
#define WIFI_PASSWORD "你的WiFi密码"       // 改成你的密码

// 服务器配置
#define SERVER_HOST "192.168.1.105"        // 改成你刚才查到的IP
#define SERVER_PORT 3000                   // 保持不变
```

然后编译上传到ESP32：

```bash
cd d:/test111/iottest
pio run --target upload
pio device monitor
```

## 常用服务器命令

在服务器控制台中可以输入：

| 命令 | 说明 |
|------|------|
| `status` | 查看服务器状态 |
| `telemetry` | 查看接收到的传感器数据 |
| `devices` | 查看已注册的设备 |
| `help` | 查看所有命令 |
| `exit` | 退出服务器 |

## 常见问题

### Q: 端口3000被占用

**解决：**
```bash
# 使用其他端口启动
ESP_MOCK_PORT=3001 node esp-mock-server.js
```

然后在config.h中修改：
```cpp
#define SERVER_PORT 3001
```

### Q: ESP32无法连接服务器

**检查：**
1. 服务器是否正在运行
2. IP地址是否正确
3. ESP32和电脑是否在同一WiFi网络
4. 防火墙是否阻止了端口3000

### Q: 服务器收不到数据

**检查：**
1. ESP32串口是否显示"数据发送成功"
2. 服务器URL是否正确
3. 网络连接是否稳定
