# 服务器控制台使用说明

## 如何打开服务器控制台

### 步骤1：启动服务器

打开命令行（CMD或PowerShell），执行：

```bash
cd d:/test111/net/HarmoAgri/tools
node esp-mock-server.js
```

### 步骤2：服务器启动成功后

你会看到类似这样的输出：

```
ESP mock server running at http://127.0.0.1:3000
Loaded config: tools/mock-config.default.json
Commands:
  help
  status
  telemetry
  devices
  ...
```

**这个窗口就是服务器控制台！保持它运行，不要关闭。**

---

## 控制台常用命令

当ESP32连接并发送数据后，你可以在控制台中输入以下命令查看数据：

### 查看传感器数据

```bash
telemetry
```

**输出示例：**
```json
{
  "temperature": 25.5,
  "humidity": 60.2,
  "light": 450,
  "co2": 780,
  "soilMoisture": 35.8,
  "timestamp": "2024-03-29T23:45:30.000Z"
}
```

### 查看已注册设备

```bash
devices
```

**输出示例：**
```json
[
  {
    "id": "esp32-sensor-001",
    "name": "ESP32环境监测站",
    "type": "SENSOR",
    "online": true,
    "status": "就绪",
    "location": "北侧棚区"
  }
]
```

### 查看服务器状态

```bash
status
```

**输出示例：**
```json
{
  "gateway": {
    "gatewayId": "gateway-001",
    "online": true,
    "lastHeartbeat": "2024-03-29T23:45:30.000Z",
    "firmwareVersion": "1.0.0",
    "connectedDeviceCount": 1
  }
}
```

### 查看告警信息

```bash
alerts
```

**输出示例：**
```json
[
  {
    "level": "WARNING",
    "message": "土壤湿度偏低",
    "source": "soil",
    "time": "2024-03-29T23:45:30.000Z"
  }
]
```

### 查看所有命令

```bash
help
```

## 完整命令列表

| 命令 | 说明 | 使用场景 |
|------|------|----------|
| `help` | 显示所有可用命令 | 忘记命令时使用 |
| `status` | 查看服务器和网关状态 | 检查服务器是否正常 |
| `telemetry` | 查看最新的传感器数据 | **最常用！查看ESP32发送的数据** |
| `devices` | 查看已注册的设备列表 | 确认ESP32是否成功注册 |
| `discovered` | 查看待添加的设备 | 设备发现功能 |
| `alerts` | 查看系统告警 | 检查是否有异常 |
| `commands` | 查看命令执行历史 | 调试命令功能 |

## 高级功能

### 手动设置传感器数据

你可以手动修改传感器数据来测试前端显示：

```bash
# 设置温度为30度
set telemetry temperature 30

# 设置湿度为70%
set telemetry humidity 70

# 设置光照为500
set telemetry light 500
```

### 模拟设备操作

```bash
# 将设备设置为开启状态
device on esp32-sensor-001

# 将设备设置为关闭状态
device off esp32-sensor-001

# 设置设备档位
device level pump-001 3
```

### 模拟故障

```bash
# 模拟网关离线
fault gateway on

# 模拟命令失败
fault commandFail on

# 恢复正常
fault gateway off
fault commandFail off
```

### 保存当前状态

```bash
# 保存到默认文件
save state

# 保存到指定文件
save state my-snapshot.json
```

### 重置服务器状态

```bash
reset
```

## 查看数据的三种方式

### 方式1：直接在控制台输入命令（推荐）

这是最简单的方式，直接在服务器运行的命令行窗口中输入命令。

```
> telemetry
{
  "temperature": 25.5,
  "humidity": 60.2,
  ...
}
```

### 方式2：使用浏览器访问API

打开浏览器，访问：

```
http://localhost:3000/telemetry/realtime
http://localhost:3000/devices
http://localhost:3000/gateway/status
```

### 方式3：使用curl命令

在另一个命令行窗口中执行：

```bash
# 查看传感器数据
curl http://localhost:3000/telemetry/realtime

# 查看设备列表
curl http://localhost:3000/devices

# 查看服务器状态
curl http://localhost:3000/gateway/status
```

## 常见问题

### Q1：输入命令后没有反应

**解决：**
- 确保光标在服务器控制台窗口中
- 输入命令后按回车键
- 检查命令拼写是否正确

### Q2：看不到ESP32发送的数据

**检查：**
1. ESP32是否成功连接WiFi
2. ESP32串口监视器是否显示"数据发送成功"
3. 服务器IP地址配置是否正确
4. 服务器是否正在运行

### Q3：数据格式不正确

**解决：**
- 检查ESP32代码中的JSON格式
- 确保使用ArduinoJson库正确序列化
- 查看串口监视器的错误信息

## 调试技巧

### 1. 实时监控数据

每5秒输入一次 `telemetry` 命令，观察数据变化：

```bash
telemetry
# 等待5秒
telemetry
# 等待5秒
telemetry
```

### 2. 检查设备注册

在ESP32启动后立即执行：

```bash
devices
```

查看设备是否成功注册。

### 3. 查看完整状态

执行 `status` 命令查看所有状态信息，包括：
- 网关状态
- 传感器数据
- 设备列表
- 故障设置
