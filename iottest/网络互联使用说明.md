# ESP32 IoT智能农业项目 - 网络互联使用说明

## 概述

本文档说明如何将ESP32传感器设备与后端服务器进行互联，实现传感器数据的实时上传和远程监控。

## 系统架构

```
┌─────────────────┐      HTTP/WebSocket      ┌─────────────────┐
│   ESP32设备     │ ◄──────────────────────► │   后端服务器     │
│   (传感器数据)   │                          │   (Node.js)     │
└─────────────────┘                          └─────────────────┘
                                                      │
                                                      │ HTTP/WebSocket
                                                      ▼
                                            ┌─────────────────┐
                                            │   前端应用       │
                                            │   (HarmonyOS)   │
                                            └─────────────────┘
```

## 配置步骤

### 1. 修改WiFi配置

编辑 `iottest/src/config.h` 文件，修改以下配置：

```cpp
// WiFi配置
#define WIFI_SSID "你的WiFi名称"           // 修改为你的WiFi名称
#define WIFI_PASSWORD "你的WiFi密码"       // 修改为你的WiFi密码

// 服务器配置
#define SERVER_HOST "192.168.1.100"        // 修改为后端服务器IP地址
#define SERVER_PORT 3000                   // 修改为后端服务器端口

// 设备配置
#define DEVICE_ID "esp32-sensor-001"       // 可自定义设备ID
#define DEVICE_NAME "ESP32环境监测站"       // 可自定义设备名称
#define DEVICE_LOCATION "北侧棚区"          // 修改为实际位置
```

### 2. 启动后端服务器

在 `net/HarmoAgri/tools/` 目录下启动模拟服务器：

```bash
# 进入工具目录
cd net/HarmoAgri/tools

# 启动模拟服务器
node esp-mock-server.js
```

服务器启动后会显示：
```
ESP mock server running at http://127.0.0.1:3000
Loaded config: tools/mock-config.default.json
```

### 3. 编译和上传ESP32代码

使用PlatformIO编译和上传：

```bash
# 进入项目目录
cd iottest

# 编译项目
pio run

# 上传到ESP32
pio run --target upload

# 监控串口输出
pio device monitor
```

## 功能说明

### 网络通信模块

新增的网络通信模块 (`network_module.h` 和 `network_module.cpp`) 提供以下功能：

1. **WiFi自动连接**
   - 开机自动连接WiFi
   - 断线自动重连
   - 连接状态指示

2. **HTTP通信**
   - 发送传感器数据到服务器
   - 注册设备到服务器
   - 获取网关状态

3. **数据格式**
   - JSON格式数据传输
   - 包含所有传感器数据
   - 时间戳自动更新

### 数据传输格式

传感器数据以JSON格式发送到服务器：

```json
{
  "temperature": 24.5,
  "humidity": 59.2,
  "light": 450,
  "tvoc": 86,
  "eco2": 780,
  "soilMoisture": 35.2,
  "soilTemperature": 23.6,
  "rainDetected": false,
  "motionDetected": false,
  "timestamp": "2024-01-01T12:00:00"
}
```

### API接口

ESP32会调用以下API接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/devices` | POST | 注册设备 |
| `/telemetry/realtime` | POST | 上传传感器数据 |
| `/gateway/status` | GET | 获取网关状态 |

## 故障排除

### 1. WiFi连接失败

**问题**：显示"WiFi连接失败"

**解决方案**：
- 检查WiFi名称和密码是否正确
- 确认WiFi信号强度
- 检查ESP32是否支持WiFi频段（2.4GHz）

### 2. 服务器连接失败

**问题**：显示"HTTP请求失败"

**解决方案**：
- 确认服务器IP地址和端口
- 检查服务器是否正在运行
- 确认ESP32和服务器在同一网络

### 3. 数据发送失败

**问题**：显示"数据发送失败"

**解决方案**：
- 检查服务器API接口是否正常
- 查看串口输出的错误信息
- 确认服务器防火墙设置

## 进阶配置

### 1. 修改数据发送间隔

在 `config.h` 中修改：

```cpp
#define DATA_SEND_INTERVAL 10000  // 改为10秒发送一次
```

### 2. 启用WebSocket（可选）

在 `config.h` 中修改：

```cpp
#define WS_ENABLED true           // 启用WebSocket
#define WS_PATH "/ws"             // WebSocket路径
```

### 3. 自定义设备信息

修改设备相关配置：

```cpp
#define DEVICE_ID "my-esp32-001"
#define DEVICE_NAME "我的监测站"
#define DEVICE_TYPE "SENSOR"
#define DEVICE_LOCATION "温室A区"
```

## 注意事项

1. **网络稳定性**
   - 建议使用稳定的WiFi网络
   - 避免网络频繁断开

2. **电源供应**
   - 确保ESP32有稳定的电源供应
   - 网络通信会增加功耗

3. **数据安全**
   - 生产环境建议使用HTTPS
   - 添加数据验证和加密

4. **服务器性能**
   - 多个设备同时连接时注意服务器负载
   - 适当调整数据发送频率
